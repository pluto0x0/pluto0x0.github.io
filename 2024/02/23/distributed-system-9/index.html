Distributed System (9)

Mutual Exclusion

Ricart-Agrawala’s Algorithm

  • enter() at process Pi
    • set state to Wanted
    • multicast “Request” <Ti,Pi><T_i, P_i><Ti​,Pi​> to all processes
      where Ti=T_i =Ti​= current Lamport timestamp at PiP_iPi​
    • wait until all processes send back “Reply”
    • change state to Held and enter the CS
  • On receipt of a Request <Tj,Pj><T_j, P_j><Tj​,Pj​> at PiP_iPi​ ( i≠ji \ne ji=j ):
    • if (state = Held) or (state = Wanted and (Ti,i)<(Tj,j)(T_i, i) < (T_j, j)(Ti​,i)<(Tj​,j) ) *// lexicographic ordering in (Tj,Pj)(T_j, P_j)(Tj​,Pj​) *
      • add request to local queue (of waiting requests)
    • else send “Reply” to PjP_jPj​
  • exit() at process Pi
    • change state to Released and “Reply” to all queued requests.

Maekawa’s Algorithm

put NNN processes in a N\sqrt{N}N​ by N\sqrt{N}N​ matrix and for each PiP_iPi​ , its voting set Vi=V_i =Vi​= row containing PiP_iPi​ + column containing PiP_iPi​ (each set size = 2N−12\sqrt{N}-12N​−1 )


voting set

  • state = Released, voted = false
  • enter() at process Pi:
    • state = Wanted
    • Multicast Request message to all processes in ViV_iVi​
    • Wait for Reply (vote) messages from all processes in ViV_iVi​ (including vote from self)
    • state = Held
  • exit() at process Pi:
    • state = Released
    • Multicast Release to all processes in ViV_iVi​ :
  • When PiP_iPi​ receives a Request from PjP_jPj​ :
    • if (state ====== Held OR voted ====== true)
      • queue Request
    • else
      • send Reply to PjP_jPj​ and set voted === true
  • When PiP_iPi​ receives a Release from PjP_jPj​ :
    • if (queue empty)
      • voted === false
    • else
      • dequeue head of queue as PkP_kPk​
      • Send Reply only to PkP_kPk​
      • voted = true
