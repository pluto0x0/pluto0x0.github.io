Distributed System (5)

Chandy-Lamport Algorithm

(Snapshot should not interfere with normal application actions, and it should not require application to stop sending messages.)

First, initiator PiP_iPi​ :

  • records its own state
  • creates a special marker message.
  • for j=1…n,j≠ij=1\ldots n, j\ne ij=1…n,j=i
    • PiP_iPi​ sends a marker message on outgoing channel cijc_{ij}cij​
    • start recording cjic_{ji}cji​

For other process PiP_iPi​ receiving a marked message from channel ckic_{ki}cki​ ,

  • if it is the first time PiP_iPi​ see marked message
    • PiP_iPi​ records its own state
    • marks the state of ckic_{ki}cki​ as “empty”
    • for j=1…n,j≠ij=1\ldots n, j\ne ij=1…n,j=i
      • PiP_iPi​ sends a marked message over cijc_{ij}cij​
      • if j≠kj\ne kj=k
        • start recording cjic_{ji}cji​
  • else
    • stop recording ckic_{ki}cki​ (the state of channel contains all messages during recording)

Any run of the Chandy-Lamport Global Snapshot algorithm creates a consistent cut.

Proof.

  • for every pair of causal messages mi→mjm_i \to m_jmi​→mj​ corresponding to process Pi,PjP_i, P_jPi​,Pj​ , if mjm_jmj​ is in the cut, then mim_imi​ is also in the cut.
  • prove by contracdition: if not, mim_imi​ was sent before PjP_jPj​ sending the marked message but arrive before it, then the cannel is not FIFO.
